# Plan 4: Collaboration + PWA + Polish

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Trip collaboration (co-editor invites), read-only share links, PWA manifest + service worker for offline itinerary viewing, and Motion-based animations for a polished final experience.

**Prerequisite:** Plan 3 complete and verified.

**Architecture:** Collaboration uses existing `trip_collaborators` table — backend middleware (`tripAccess.js`) already distinguishes owner/editor/viewer roles. Share links use a public token route that returns read-only trip data. PWA: Vite PWA plugin generates service worker; caches app shell + per-trip itinerary JSON. Motion animations added as a final pass — staggered stop card reveals, co-pilot slide-up spring, day tab cross-fade.

**Tech Stack:** vite-plugin-pwa (Workbox), framer-motion (Motion), QR code library for share link display

**Design spec:** `docs/superpowers/specs/2026-04-23-trippy-design.md` — §7.8 Collaboration, §8.7 Motion, §11 PWA
